    Documentación de Funciones para Procesos ETL en Data Warehouse¶

    Este documento proporciona una guía detallada de las funciones contenidas en el archivo Funciones.py, utilizadas para procesos ETL (Extracción, Transformación y Carga) en un sistema de Data Warehouse. A continuación, se describe cada función, las librerías utilizadas, un diagrama de los procesos principales, y un ejemplo del código fuente. Estas funciones están organizadas en categorías clave según su propósito en el flujo de trabajo.

    Librerías Utilizadas¶

    • pandas: Manipulación y transformación de datos.
    • os: Interacción con el sistema operativo.
    • sqlalchemy: Conexión y manejo de bases de datos SQL.
    • time y datetime: Manejo y registro de tiempos de ejecución.
    • logging: Registro de eventos para monitoreo y depuración de procesos ETL.

    Categorías de Funciones y Descripción General¶

    1. Conexiones a Bases de Datos¶

    Estas funciones generan motores y cadenas de conexión según los detalles de configuración, permitiendo conectarse a distintas bases de datos.

    • Funciones Incluidas: Conexion_dwh, Conexion_Minerva, Conexion_neith, obtener_conexion
    • Ejemplo de Código: 
      def obtener_conexion(nombre_bd):
    config = configuraciones_bd.get(nombre_bd)
    cadena_conexion = f"mysql+pymysql://{config['usuario']}:{config['contraseña']}@{config['host']}/{config['base_de_datos']}"
    return cadena_conexion

    2. Estandarización de Direcciones¶

    Estas funciones estandarizan y marcan direcciones utilizando abreviaciones comunes, simplificando las comparaciones y el análisis de direcciones.

    • Funciones Incluidas: estandarizar_direccion, marcar_direcciones_estandarizadas
    • Descripción:
    • estandarizar_direccion(direccion): Normaliza la dirección eliminando caracteres especiales y aplicando abreviaciones.
    • marcar_direcciones_estandarizadas(df, columna_direccion_limpia): Etiqueta las direcciones estandarizadas, facilitando la validación de datos.
    • Ejemplo de Código: 
      def estandarizar_direccion(direccion):
    ABREVIACIONES = {"CALLE": "CL", "CARRERA": "CR", "TRANSVERSAL": "TV", "AVENIDA": "AV"}
    for palabra, abrev in ABREVIACIONES.items():
        direccion = direccion.replace(palabra, abrev)
    return direccion.strip()

    3. Manejo de Duplicados y Validaciones¶

    Estas funciones identifican y eliminan duplicados en los DataFrames y generan archivos de trazabilidad, útiles para auditoría y calidad de datos.

    • Funciones Incluidas: StoreDuplicated, RemoveDuplicated, RemoveErrors
    • Descripción:
    • StoreDuplicated: Identifica y almacena registros duplicados en un archivo CSV.
    • RemoveDuplicated: Ordena y elimina duplicados basándose en una columna específica.
    • RemoveErrors: Elimina registros con inconsistencias entre fechas de afiliación y retiro.
    • Ejemplo de Código: 
      def StoreDuplicated(columId, ColumnsToCompare, table, NombreArchivo):
    subsetDuplicate = table.columns.tolist() if columId == 'N/A' else ColumnsToCompare
    duplicated_rows = table[table.duplicated(subset=subsetDuplicate, keep=False)]
    duplicated_rows.to_csv(NombreArchivo + '.csv', index=False)

    4. Limpieza y Normalización de Textos¶

    Estas funciones normalizan cadenas de texto para asegurar la consistencia en los nombres de columnas y valores textuales en general.

    • Funciones Incluidas: normalize, RemovePreposition, CorrectionColumnName, convertir_columnas_mayusculas
    • Descripción:
    • normalize: Elimina acentos en cadenas de texto.
    • RemovePreposition: Elimina preposiciones en los nombres de columnas.
    • CorrectionColumnName: Genera nombres de columnas basados en su descripción.
    • convertir_columnas_mayusculas: Convierte los nombres de columnas a mayúsculas.
    • Ejemplo de Código: 
      def normalize(s):
    replacements = (("á", "a"), ("é", "e"), ("í", "i"))
    for a, b in replacements:
        s = s.replace(a, b)
    return s.upper()

    5. Carga de Tablas y Almacenamiento en el Data Warehouse¶

    Estas funciones administran el almacenamiento y la carga de datos en el Data Warehouse, incluyendo la opción de registrar el tiempo de carga.

    • Funciones Incluidas: cargar_tablas, guardar_en_dwh, log_tiempo
    • Descripción:
    • guardar_en_dwh: Almacena un DataFrame en el Data Warehouse.
    • cargar_tablas: Ejecuta una consulta SQL y carga los resultados en un DataFrame.
    • log_tiempo: Registra el tiempo de ejecución de los procesos.
    • Ejemplo de Código: 
      def guardar_en_dwh(df, tabla, conexion):
    df.to_sql(tabla, con=conexion, if_exists='replace', index=False)

    6. Configuración de Logger y Herramientas Adicionales¶

    Configura el logger para registrar mensajes y permite verificar la disponibilidad de las funciones con un mensaje de prueba.

    • Funciones Incluidas: setup_logger, testfunciones
    • Descripción:
    • setup_logger: Configura el logger para registrar en archivo y consola.
    • testfunciones: Prueba la importación de funciones y muestra la fecha y hora actual.
    • Ejemplo de Código: 
      def setup_logger(log_filename='app.log'):
    logger = logging.getLogger()
    logger.setLevel(logging.DEBUG)
    file_handler = logging.FileHandler(log_filename)
    logger.addHandler(file_handler)
    return logger

    Conclusión¶

    El archivo Funciones.py contiene herramientas esenciales para procesos ETL, organizadas en categorías de conectividad, transformación, almacenamiento y trazabilidad de datos. Cada función desempeña un rol específico, asegurando calidad, consistencia y eficiencia en el procesamiento de datos para su posterior análisis en un sistema de Data Warehouse.

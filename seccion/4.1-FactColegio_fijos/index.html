    4.1 Fact Colegio fijos¶

    Introducción¶

    El proceso ETL para FactColegio_fijos extrae y transforma datos de las tablas colegio12 y colegio13, correspondientes a información de estudiantes y acudientes, respectivamente. La transformación incluye la consolidación de los datos en un DataFrame único, la estandarización de nombres y la limpieza de duplicados. Una vez estructurados los datos, se verifica su existencia en la tabla DimDatosFijos del Data Warehouse (DWH). Los registros que no existen se almacenan en el DWH, asegurando la integridad y unicidad de la información de afiliados en la base de datos.

    Consultas SQL para Cargar Datos de Estudiantes y Acudientes¶

    Este código define dos consultas SQL (colegio12 y colegio13) para extraer datos de estudiantes y acudientes desde las tablas colegio12 y colegio13. Ambas consultas construyen el campo ID_AFILIADO combinando los valores coddoc y numdoc, y crean RAZON_SOCIAL concatenando nombres y apellidos.

    La estructura también incluye FECHA_NACIMIENTO, SEXO, CODDOC, DOCUMENTO, y los nombres individuales (PRIMER_NOMBRE, SEGUNDO_NOMBRE, PRIMER_APELLIDO, SEGUNDO_APELLIDO). En colegio13, FECHA_NACIMIENTO se deja como NULL, dado que esta información podría no estar disponible para acudientes.

    Finalmente, las consultas se registran en qr_structure, y el logger anota el inicio de la lectura de queries en el proceso ETL.

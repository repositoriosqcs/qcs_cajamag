    2.1 Fact Aportes Emp Afiliadas¶

    Intoducción¶

    El proceso FactAportesEmpAfiliadas consiste en un pipeline ETL que extrae datos de aportes empresariales desde las tablas xml4.xml4c085 y subsidio.subsi11 en la base de datos MINERVA, transformándolos para integrar información sobre valores pagados, intereses, número de trabajadores aportantes y otros datos clave. Los registros extraídos son procesados para eliminar duplicados, limpiar valores inconsistentes y crear claves únicas, asegurando una estructura consolidada y sin redundancias. Finalmente, los datos transformados se cargan en el data warehouse (DWH) en la tabla BD_Fact_HistoricoAportesEmpAfiliadas para su almacenamiento y análisis, documentando el proceso en un archivo de log para control y monitoreo de eventos.

    Definición de Consulta SQL para Datos de Aportes¶

    En este código, se define una consulta SQL dentro del diccionario qr_structure que extrae y organiza información sobre aportes en la tabla aportes. La consulta combina datos de las tablas xml4.xml4c085 y subsidio.subsi11 utilizando un LEFT JOIN, basado en la coincidencia de columnas nit y periodo.

    En la primera subconsulta (p1), se generan identificadores únicos (id) concatenando tipide y nit, y se filtran los registros dentro de los últimos 18 meses. Esta subconsulta agrupa los valores de aportes y de intereses (valorpag y valorInt) por periodo, nit, y tipapo. La segunda subconsulta (p2) agrega valores de aportes (valorcon), trabajadores aportantes (trabaapo), y clasifica cada registro como "Empresa" en la columna Tipo.

    El resultado de la consulta unificada (b) permite un análisis de los aportes de cada empresa o entidad, facilitando el seguimiento de pagos, periodos referenciados y valores de aportes e intereses. La estructura resultante se almacena en df_structure para uso posterior en análisis o almacenamiento en el sistema de datos.

    #Lista de querys
qr_structure = {
    "aportes":'''
select b.id, b.nit, b.fecpag, b.tipapo, b.periodorefer, b.valorcon, b.valorpag, b.trabaapo, b.Tipo, b.valorInt from
(select p1.*, p2.* from
(select CONCAT(CAST(CASE WHEN tipide = 1 THEN 'CC' WHEN tipide = 3 THEN 'RC' WHEN tipide = 4 THEN 'CE' WHEN tipide = 6 THEN 'PA' WHEN tipide = 7 THEN 'NI'            
ELSE tipide END AS CHAR), nit) as id, nit, periodo as fecpag, tipapo, SUM(apomes)  as valorpag, SUM(valint)  as valorInt
from xml4.xml4c085 where (CONCAT(SUBSTRING(periodo, 1, 4), '-', SUBSTRING(periodo, 5, 2), '-01') <= CURRENT_DATE()
AND CONCAT(SUBSTRING(periodo, 1, 4), '-', SUBSTRING(periodo, 5, 2), '-01') >= cast(DATE_SUB(CURRENT_DATE(), INTERVAL 18 MONTH) as CHAR))
group by periodo, nit, tipapo) as p1

left join

(select (CONCAT(SUBSTRING(fecpag, 1, 4), SUBSTRING(fecpag, 6, 2))) COLLATE utf8mb4_general_ci as periodo,
 periodo as periodorefer, nit as nit_2, SUM(valcon) as valorcon, SUM(traapo) as trabaapo, 'Empresa' as Tipo
from subsidio.subsi11 group by  (CONCAT(SUBSTRING(fecpag, 1, 4), SUBSTRING(fecpag, 6, 2))) COLLATE utf8mb4_general_ci, nit ) as p2
on p1.nit = p2.nit_2 and p1.fecpag =p2.periodo
) as b
;
'''

    Validador de Campos Repetidos y Limpieza de Duplicados¶

    Este bloque de código valida y elimina registros duplicados en cada DataFrame de df_structure. Primero, se definen las columnas a comparar (ColumnsToCompare), excluyendo la columna id, y se llama a la función StoreDuplicated para almacenar cualquier registro duplicado en un archivo con el prefijo trazaDuplicados_. Luego, se eliminan los duplicados dentro de cada DataFrame usando drop_duplicates() para conservar solo las filas únicas.

    Cada validación de tabla se registra en el logger, y al final, se muestra en el log el tiempo total de ejecución del validador de duplicados, permitiendo monitorear la eficiencia del proceso de limpieza y la trazabilidad de los datos procesados.

    #Validador de campos repetidos 
validador_time = time.time()
for ky in list(df_structure.keys()):   
    ColumnsToCompare = [x for x in df_structure[ky].columns.tolist() if x not in [ 'id' ] ]
    StoreDuplicated('N/A' , ColumnsToCompare, df_structure[ky], r'trazaDuplicados_' + ky )
    #Limpieza inicial quitar duplicados
    df_structure[ky] = df_structure[ky].drop_duplicates()
    logger.info('VALIDADOR TABLA: '+ ky)
logger.info(f'VALIDADOR DUPLICADOS --- {time.time() - validador_time:.2f} seconds ---')

    Transformación y Estandarización de Datos en df_structure¶

    Este bloque de código aplica transformaciones en df_structure para estandarizar los datos. Primero, convierte todas las columnas de texto a mayúsculas, elimina espacios en blanco y reemplaza valores "NAN" y "NONE" con NaN. Luego, concatena todos los DataFrames en df_structure en uno solo y renombra la columna fecpag como periodo.

    Se agrega la columna KeyHistoricos, que combina periodo, nit, y los primeros tres caracteres de Tipo para crear una clave única. Las columnas se renombran en mayúsculas, y algunos nombres específicos como ID, TIPAPO, y PERIODOREFER se actualizan para una mayor claridad (por ejemplo, ID se convierte en ID_AFILIADO). Además, se crea una columna ID_REGISTRO con valores secuenciales para identificar cada fila de forma única. Finalmente, se reorganizan las columnas en el orden especificado en columnas_ordenadas para obtener un DataFrame estandarizado (df_final) listo para análisis o carga en un sistema de almacenamiento.

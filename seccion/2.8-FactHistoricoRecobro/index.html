    2.8 Fact Historico Recobro¶

    Introducción¶

    El proceso ETL para Fact_HistoricoRecobro se enfoca en extraer, transformar y cargar datos históricos de subsidios y beneficiarios desde la base de datos subsidio a un Data Warehouse (DWH). Utilizando consultas SQL, se combinan las tablas subsi219 y subsi220 para reunir información relevante sobre los subsidios otorgados y los detalles de los beneficiarios en los últimos 18 meses. La transformación incluye el procesamiento de datos numéricos y la generación de claves únicas para identificar a los beneficiarios y su historial. Al final, se carga el DataFrame resultante en la tabla BD_Fact_HistoricoRecobro del DWH, registrando cada paso del proceso en el log para garantizar un flujo de trabajo supervisado y eficiente.

    Consulta de Subsidios y Beneficiarios con Filtrado de Período¶

    Esta consulta SQL extrae datos de beneficiarios y subsidios desde las tablas subsi219 y subsi220 en una base de datos de subsidios. La tabla subsi219 (t1) contiene información del subsidio, mientras que subsi220 (t2) proporciona detalles adicionales del beneficiario, como valabo y periodo. Se realiza una unión INNER JOIN entre ambas tablas en el campo idereg de t1 y ide178 de t2, vinculando los subsidios con los beneficiarios correspondientes.

    Se filtran los registros donde t1.tipsub es igual a 'S', y periodo se convierte en una fecha en formato '%Y-%m-%d'. Solo se incluyen los registros en un rango de 18 meses desde la fecha actual. La consulta también genera una columna ID_AFILIADO, creada a partir de la concatenación de coddoc y cedtra para identificar de forma única a cada beneficiario.

    # Lista de querys
qr_structure = {
    "query": """
    select 
        t1.tipsub,
        t1.cedtra, 
        t1.valdes, 
        t2.valabo, 
        t1.estado, 
        t1.idereg, 
        t1.categoria, 
        t2.ide178, 
        t2.cedtra as cedtra2, 
        t2.codben, 
        t2.periodo,
        CONCAT(t1.coddoc, t1.cedtra) as ID_AFILIADO
    from subsidio.subsi219 as t1
    INNER JOIN subsidio.subsi220 as t2
    ON t1.idereg = t2.ide178
    where t1.tipsub = 'S'
    AND STR_TO_DATE(CONCAT(SUBSTRING(t2.periodo, 1, 4), '-', SUBSTRING(t2.periodo, 5, 2), '-01'), '%Y-%m-%d')
    BETWEEN DATE_SUB(CURDATE(), INTERVAL 18 MONTH) AND CURDATE();
    """

    Transformación de Datos y Normalización de Columnas en DataFrame de Subsidios¶

    Este código aplica varias transformaciones y validaciones en el DataFrame df para asegurar que los datos de subsidios y beneficiarios estén en el formato correcto y se integren con facilidad. Se convierte valabo y valdes a tipo float para manejar valores monetarios, y codben se convierte a texto para evitar errores de formato en operaciones posteriores.

    Una nueva columna KeyHistorico se crea como identificador único, combinando periodo, codben, y el sufijo "Ben". La columna Comparacion compara cedtra y cedtra2 y, si todos los valores coinciden (True), se eliminan las columnas cedtra2 y Comparacion, limpiando el DataFrame de datos redundantes.

    Finalmente, se estandarizan los nombres de las columnas con el diccionario _names mediante la función convertir_columnas_mayusculas, renombrando codben a CODIGO_BENEFICIARIO y cedtra a CEDULA_TRABAJADOR. El método columns.tolist() permite ver el listado final de nombres de columnas en df_final.

    df = df_structure['query']

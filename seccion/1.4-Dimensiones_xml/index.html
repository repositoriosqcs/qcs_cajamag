    1.4 Dimensiones xml¶

    Intoducción¶

    Este proceso ETL (Extract, Transform, Load) tiene como finalidad extraer datos de varias tablas relacionadas con información demográfica y de beneficios de bases de datos xml4 y consolidarlos en el Data Warehouse (DWH). A través de SQL y Python, se aplican transformaciones para limpiar y estandarizar los datos antes de cargarlos en tablas dimensionales y de hechos en el DWH.

    Las tablas de entrada incluyen xml4b093, que contiene estados civiles; xml4b005, que proporciona detalles de género; xml4b047, que almacena información de nivel educativo; y xml4c087, que se utiliza para contar los beneficios únicos por trabajador, permitiendo obtener una tabla de hechos. La tabla xml4b008 aporta información de categorías relevantes. Estas fuentes ofrecen una base diversa de datos que permite construir dimensiones clave para el análisis de poblaciones y beneficios en el DWH.

    Las tablas de salida se estructuran en el DWH como:

    • BD_Dim_Estado_Civil: Define los distintos estados civiles.

    • BD_Dim_Genero: Categoriza los géneros.

    • BD_Dim_Nivel_Educativo: Clasifica el nivel educativo de los registros.

    • BD_Fact_Beneficio_Por_Trabajador: Tabla de hechos que cuenta los beneficios únicos asociados a cada trabajador.

    • BD_Dim_Categoria: Estructura y organiza las categorías de afiliación.

    Definición de consultas SQL¶

    Este código define una consulta SQL en un diccionario (qr_structure) que extrae información de la tabla subsi09 y la combina con la tabla Oportunidad mediante un LEFT JOIN. Se incluyen cálculos condicionales para determinar la clasificación del subsidio (tipsub como 'Rural', 'Urbano' o 'No definido') y para identificar si los pagos fueron oportunos o no (Oportunidad). Esto permite analizar tanto la oportunidad de los pagos como el tipo de subsidio entregado a cada beneficiario (codben).

    Iteración y Conversión de Nombres de Columnas a Mayúsculas¶

    Este fragmento de código recorre cada par key y df en el diccionario df_structure, donde key representa la clave del DataFrame y df el DataFrame en sí. Dentro del bucle, se convierte cada nombre de columna a mayúsculas para estandarizar la estructura de datos, asignando a df.columns una lista de nombres en mayúsculas generada mediante una comprensión de lista. Finalmente, se imprime el nombre de la clave (key) junto con la lista de nombres de columnas actualizados, permitiendo verificar que la conversión se realizó correctamente para cada DataFrame.

    # Iterar sobre los valores (DataFrames) en el diccionario
for key, df in df_structure.items():
    df.columns = [col.upper() for col in df.columns]
    print(f"{key}: {df.columns.tolist()}")

    Registro de Almacenamiento de Tablas en Data Warehouse (DWH)¶

    En este proceso de almacenamiento en el DWH, cada DataFrame dentro del diccionario df_structure se guarda en una tabla correspondiente en la base de datos. La función guardar_en_dwh, configurada con multiple=True y if_exists='replace', permite iterar sobre los DataFrames y guardar cada uno bajo el nombre de tabla especificado en dim_names, sobrescribiendo los datos existentes.

    Para cada tabla, el logger registra el inicio de la conexión a la base DWH y la confirmación exitosa de la conexión. A continuación, se detallan los mensajes de log para cada DataFrame, indicando su nombre de origen y el nombre con el que fue almacenado en la base de datos, lo que facilita el seguimiento del proceso. El tiempo total de ejecución del almacenamiento fue de 9.96 segundos, y el logger confirma la finalización del proceso. Este registro permite verificar la efectividad del almacenamiento, detectar posibles problemas y asegurar que cada tabla se guardó correctamente en el DWH.

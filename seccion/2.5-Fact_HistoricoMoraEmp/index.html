    2.5 Fact Historico Mora Emp¶

    Introducción¶

    El proceso ETL para Fact_HistoricoMoraEmp permite la extracción, transformación y carga de datos de morosidad de empresas desde la base de datos Minerva hacia un Data Warehouse (DWH). Este flujo inicia con la configuración del entorno, incluyendo bibliotecas y funciones necesarias para el manejo y transformación de datos. Luego, se realiza una consulta en la base Minerva para obtener datos de la tabla xml4.xml4c070, que registra valores en mora, periodos en mora y estados de notificación para cada empresa (nit). La consulta incluye un LEFT JOIN con la tabla subsidio.subsi173 para verificar el estado de notificación de cada empresa en los últimos 18 meses. Posteriormente, se transforman los datos, estandarizando nombres de columnas, creando claves únicas, y clasificando los periodos de mora en rangos. Finalmente, el resultado se almacena en la tabla BD_Fact_HistoricoMoraEmp del DWH, sobrescribiendo cualquier dato previo, y el proceso completo se registra en el log para monitoreo y análisis de rendimiento.

    Consulta SQL para Análisis de Morosidad y Notificaciones de Empresas¶

    Este código define una consulta SQL en el diccionario qr_structure, que extrae datos relacionados con la morosidad de empresas desde la tabla xml4.xml4c070. La consulta calcula el valor total en mora (salmor) y el número de periodos en mora (permor) por cada empresa (nit). Además, agrega una columna Notificacion para indicar si la empresa fue notificada, comparando con registros de la tabla subsidio.subsi173. La consulta filtra datos entre los últimos 18 periodos, y agrupa los resultados por periodo y empresa. Esto permite evaluar tanto el valor en mora como el estado de notificación de cada empresa en los periodos seleccionados.

    #Lista de querys
qr_structure = {
    "query": """
        select 
            m1.periodo as periodo_actualizacion, 
            m1.nit, 
            m1.coddoc,
            SUM(m1.salmor) as valormor, 
            m1.permor as NumPerMora, 
            'Empresa' as Tipo,  
            if(m2.nit is null, 0, 1) as Notificacion, 
            m1.razsoc 
        from 
            xml4.xml4c070 as m1 
        LEFT JOIN (
            select 
                perfin as periodo, 
                nit 
            from 
                subsidio.subsi173 
            group by 
                periodo, nit 
            order by 
                nit
        ) as m2 
        ON 
            m1.nit = m2.nit 
            AND m1.periodo = m2.periodo 
        where 
            STR_TO_DATE(CONCAT(SUBSTRING(m1.periodo, 1, 4), '-', SUBSTRING(m1.periodo, 5, 2), '-01'), '%Y-%m-%d')
            BETWEEN DATE_FORMAT(DATE_SUB(CURDATE(), INTERVAL 18 MONTH), '%Y-%m-01')
            AND DATE_FORMAT(CURDATE(), '%Y-%m-01')
        group by 
            m1.periodo, m1.nit;    
    """
}

    Conversión y Unión de DataFrames para Documentos de Morosidad¶

Posteriormente, utiliza un merge con una unión left sobre coddoc y COD_SUPERSUBSIDIO, creando un nuevo DataFrame df_merged que incorpora información de ambos conjuntos de datos. Este proceso permite consolidar los datos de morosidad con los códigos de documentos y verificar la correcta combinación de datos.

    Transformaciones del DataFrame para Análisis de Morosidad¶

Luego, se crean las columnas KeyHistoricos, PeriodosMoraRango, y KeyConvenio, que proporcionan claves únicas y rangos de periodos en mora. La columna per_act_fecha formatea la fecha de actualización a 01/MM/AAAA y la convierte a tipo fecha para facilitar análisis temporales. Finalmente, se eliminan columnas innecesarias y se reorganiza el DataFrame en un orden específico, dejando sólo las columnas clave para el análisis.

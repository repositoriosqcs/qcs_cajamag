    Optimización del Proceso de Datos en un Data Warehouse: Descripción del Bloque 5.X¶

    Introducción¶

    Este documento presenta una descripción detallada de los bloques de datos que intervienen en el proceso de carga, transformación y almacenamiento dentro de un Data Warehouse (DWH). Estos bloques tienen un propósito específico orientado a mantener la integridad y disponibilidad de los datos provenientes de diversas fuentes, como las entidades relacionadas con instituciones educativas y el servicio social. Estas actividades incluyen la importación de librerías, el uso de funciones especializadas y la implementación de código que permite manipular la información de manera eficiente y precisa.

    Objetivos¶

    1. Proporcionar una estructura organizada y clara para la integración de datos relacionados con estudiantes, acudientes, matrículas, grados y servicio social.
    2. Garantizar la calidad de los datos mediante la aplicación de transformaciones y validaciones apropiadas.
    3. Automatizar el proceso de carga al DWH utilizando librerías y funciones especializadas.
    4. Documentar y explicar el uso de los bloques de datos, sus funciones y cómo estos contribuyen al objetivo general del Data Warehouse.

    Descripción de los Bloques de Datos¶

    1. FactServicioSocial_Fijos (Bloque 5.1)¶

    • Tabla utilizada: FactServicioSocial_Fijos
    • Descripción: Este bloque gestiona datos personales relacionados con el servicio social, integrando información desde dos fuentes diferentes (nsijec.personas y saipi.personas). Se extraen datos clave como la identificación, nombres, apellidos, y detalles de dirección de las personas involucradas. Los registros de ambas fuentes se combinan y se estructuran para garantizar consistencia. Además, se realiza una limpieza y estandarización de las direcciones para asegurar la calidad de los datos antes de su integración al Data Warehouse.

    2. FactServicioSocial (Bloque 5.2)¶

    • Tablas utilizadas: FactServicioSocial
    • Descripción: Este bloque se encarga de la integración de datos más complejos relacionados con el servicio social, incluyendo información de estudiantes, acudientes, valoraciones y registros adicionales. Utiliza consultas SQL para extraer información desde diversas tablas (estudiantes, personas_saipi, acudientes, registros, valoracion). Se aplican técnicas de limpieza y estandarización, así como validación de duplicados. Los datos se cargan en la base de datos DWH para permitir un análisis detallado de los servicios sociales proporcionados, mejorando así la visibilidad y el control sobre la gestión de estos servicios.

    Conclusión¶

    El proceso de carga y transformación de datos en un Data Warehouse es esencial para garantizar la disponibilidad de información confiable y consistente. Los bloques descritos en este documento desempeñan un papel crucial en este proceso, gestionando los datos de las instituciones educativas y de servicios sociales. La implementación de funciones personalizadas y el uso de librerías especializadas optimizan el flujo de datos, minimizan errores y maximizan la eficiencia del sistema de información.

    Un DWH bien diseñado y gestionado proporciona una base sólida para la toma de decisiones empresariales basadas en datos precisos y consistentes, lo cual es esencial para el éxito en un entorno de negocios competitivo y dinámico.

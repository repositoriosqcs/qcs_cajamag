    5.1 Fact Servicio Social Fijos¶

    Introducción¶

    El proceso ETL FactServicioSocial_Fijos extrae y transforma datos de personas desde dos fuentes (nsijec.personas y saipi.personas). Este flujo garantiza la integración de información clave de identificación en el Data Warehouse (DWH), especialmente en la tabla DimDatosFijos. Cada registro se verifica para asegurar su unicidad, y aquellos que no existen en el DWH se almacenan, asegurando una estructura de datos consolidada y uniforme.

    Definición de Consultas SQL para la Carga de Datos de Personas¶

    Este código establece consultas SQL en qr_structure y qr_structureNeith para extraer datos de personas desde dos fuentes diferentes (nsijec.personas y saipi.personas). Estas consultas recuperan información relevante como ID_AFILIADO, RAZON_SOCIAL, FECHA_NACIMIENTO, SEXO, CODDOC, DOCUMENTO, y nombres y apellidos, asegurando que los campos requeridos en required_columns estén presentes en ambas estructuras. Esto permite integrar y estandarizar datos personales de distintas fuentes dentro del Data Warehouse, facilitando el análisis y reportes de manera unificada.

    # Definir las consultas SQL para cargar datos y asegurarse de crear la estructura con los campos requeridos

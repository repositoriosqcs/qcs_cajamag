    3.1 Fact Transacciones Ventas¶

    Introducción¶

    El proceso ETL para FactTransaccionesVentas unifica y transforma datos de ventas de servicios provenientes de múltiples tablas (servi21, servi20 y servi03) de la base de datos servicios, cargándolos en una única tabla de ventas en el Data Warehouse (DWH). La extracción de registros de venta está limitada a los últimos 18 meses, asegurando que solo se procesen transacciones recientes. Se realizan transformaciones de datos, como la normalización de valores faltantes y creación de claves únicas de afiliados, que optimizan la estructura de los datos para análisis posteriores. Finalmente, los datos se cargan en la tabla BD_Fact_Ventas_Servicios en el DWH, registrando cada etapa del proceso en un log para asegurar trazabilidad y control de calidad.

    Consulta de Ventas de Servicios y Detalles de Beneficiarios¶

    Esta consulta SQL extrae información de ventas de servicios de la tabla servi21 en combinación con datos adicionales de las tablas servi20 y servi03. La unión se realiza entre servi21 y servi20 usando los campos marca y documento, y se limitan los resultados a las ventas en un período de 18 meses desde la fecha actual.

    El resultado incluye detalles de cada venta, como DOCUMENTO_VENTA, MARCA_VENTA, PERIODO, FECHA_VENTA, TIPO_BENEFICIARIO, CATEGORIA_VENTA, y otros campos descriptivos de los servicios y su costo (VALOR_SERVICIO, VALOR_DESCONTADO, VALOR_COSTO, VALOR_SUBSIDIO). La tabla servi03 se usa para obtener descripciones de categorías, mientras que servi20 aporta detalles de estado y fechas.

    Este código asegura la configuración para ejecutar la consulta y cargar los datos en df_structure para su posterior procesamiento y análisis.

    Validación y Limpieza de Datos en Tablas de Ventas y Dimensión de Datos Fijos¶

    Este bloque de código realiza validaciones, transformaciones y limpieza de datos en el diccionario df_structure, que contiene los resultados de consultas sobre ventas de servicios. En la primera etapa, se verifica si existen duplicados en las tablas usando StoreDuplicated, que genera un archivo de trazabilidad de duplicados si se encuentran registros repetidos. Luego, se eliminan duplicados directamente de las tablas para asegurar la integridad de los datos.

    En la segunda etapa, se normalizan y limpian los datos en cada tabla de df_structure transformando los nombres de columnas y los valores de las columnas de tipo texto a mayúsculas. Además, se eliminan los espacios en los extremos y se reemplazan las cadenas "NAN" o "NONE" por valores NaN de NumPy, asegurando consistencia en los valores faltantes.

    La última parte define una consulta qr_structure_datos_fijos para la tabla BD_Dim_Datos_Fijos, que se usa para obtener datos fijos, como códigos y documentos, que complementan el análisis de ventas. El resultado de esta consulta se cargará en df_structure_datos_fijos para su procesamiento posterior.

    #Validador de campos repetidos

    #Limpieza inicial quitar duplicados

    Eliminación de Duplicados y Normalización de Datos en Ventas de Servicios¶

    Este bloque de código realiza uniones y transformaciones en los datos de ventas de servicios y documentos en df_structure. Primero, elimina duplicados en df_structure_datos_fijos['BD_Dim_Datos_Fijos'] para conservar únicamente el último registro de cada documento, garantizando que solo se mantenga la información más reciente de cada documento.

    A continuación, df_ventas y df_datos_fijos se combinan mediante un join en CODBEN y DOCUMENTO, proporcionando el tipo de documento al conjunto de datos de ventas. Luego, se crea un identificador único ID_AFILIADO concatenando CODDOC y CODBEN, y se reorganizan las columnas para que ID_AFILIADO se ubique en la quinta posición, mejorando la legibilidad de los datos.

    Para asegurar consistencia en los tipos de datos, se rellenan valores faltantes en CONSECUTIVO_VENTA, EDAD, y NUMERO_APERTURA con ceros. Posteriormente, estos campos se convierten a tipo int para mantener la integridad de los datos. Finalmente, df_structure['BD_Fact_Ventas_Servicios'] se actualiza con el DataFrame df_joined, listo para el análisis o carga en la base de datos.

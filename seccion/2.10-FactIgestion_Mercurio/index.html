    2.10 Fact Igestion Mercurio¶

    Introducción¶

    El proceso ETL para Fact_Igestion_Mercurio permite consolidar datos de afiliación provenientes de múltiples fuentes (SGI, Mercurio Beneficiarios, Empresas y Trabajadores) en un único Data Warehouse (DWH). Este proceso extrae registros de radicados de afiliación, transformándolos en un formato estandarizado, y genera claves únicas para cada registro, asegurando una integración y análisis eficientes. Las transformaciones incluyen normalización de tipos de documentos, generación de períodos de referencia y claves de historial. Finalmente, los datos se cargan en la tabla BD_Fact_Igestion_Mercurio del DWH y se registran los pasos en un log para asegurar trazabilidad y rendimiento.

    Importación de librerías y funciones¶

    Este script establece el entorno inicial para un proceso de ETL (Extracción, Transformación y Carga) mediante el uso de SQLAlchemy y Pandas, facilitando la conexión a bases de datos y la manipulación avanzada de datos. Se inicia un cronómetro con time para medir la duración de los procesos, útil para evaluar el rendimiento. A través de sys.path, se habilita la carga de funciones específicas desde un módulo externo en la carpeta funciones, importando métodos como convertir_columnas_mayusculas, guardar_en_dwh, cargar_tablas, obtener_conexion, testfunciones y setup_logger. Estos métodos proporcionan capacidades para transformar y guardar datos en el Data Warehouse, realizar conexiones de base de datos, ejecutar pruebas y configurar registros de eventos. Así, el entorno queda listo para ejecutar un flujo estructurado y monitoreado de integración y transformación de datos.

    Definición de consultas SQL¶

    El código define una consulta SQL que combina datos de varias tablas en un único resultado, utilizando una unión de tipo UNION ALL para consolidar registros de afiliación desde diferentes fuentes de la base de datos dwh. La consulta extrae información relevante, incluyendo Fecha, TIPDOC (tipo de documento), Documento, Radicado, y una columna Fuente que indica la procedencia de cada registro (Igestion_SGI, Mercurio_Beneficiarios, Mercurio_Empresas, o Mercurio_Trabajadores). Esto permite unificar en una misma estructura registros provenientes de distintas tablas, facilitando así el análisis y tratamiento de datos relacionados con procesos de afiliación. Además, se registra la ejecución de la consulta en logger.info para control y trazabilidad.

    Transformación y Enriquecimiento del DataFrame para Análisis de Afiliación¶

    Este código aplica una serie de transformaciones y enriquecimientos al DataFrame df con el fin de preparar los datos para un análisis estructurado. Primero, se convierte la columna Fecha al tipo datetime para manipulación de fechas, y luego se crea la columna Periodo en el formato YYYYMM, derivando año y mes de la columna Fecha. También se define la columna tipide, asignando códigos numéricos a valores específicos en la columna TD mediante una función map_tipide, lo que permite estandarizar los diferentes tipos de documentos.

    Tras esta normalización, Periodo y tipide se convierten a tipo string para alinearse con la salida deseada, y se genera la columna KeyHistorico, concatenando Periodo, tipide y Documento para formar una clave única que identifica cada registro. Por último, la función convertir_columnas_mayusculas estandariza los nombres de las columnas a mayúsculas, asegurando consistencia en el formato final del DataFrame df_final, ideal para futuras consultas o carga en un sistema de almacenamiento.

    df = df_structure['query']

# Cambiar el tipo de la columna 'Fecha' a datetime
df['Fecha'] = pd.to_datetime(df['Fecha'], format='%Y-%m-%d')

# Agregar la columna 'Periodo' con el formato YYYYMM
df['Periodo'] = df['Fecha'].dt.year * 100 + df['Fecha'].dt.month

# Definir la columna 'tipide' con la lógica condicional
def map_tipide(td):
    if td == "CC":
        return 1
    elif td == "CD":
        return 8
    elif td == "CE":
        return 4
    elif td in ["NI", "NIT"]:
        return 7
    elif td == "NP":
        return 5
    elif td == "PA":
        return 6
    elif td in ["PE", "PEP", "PP", "TE"]:
        return 9
    elif td in ["PPT", "PT"]:
        return 15
    elif td == "RC":
        return 3
    elif td == "TI":
        return 2
    else:
        return td

df['tipide'] = df['TD'].apply(map_tipide)

# Convertir la columna 'Periodo' y 'tipide' a string (similar a type text en Power Query)
df['Periodo'] = df['Periodo'].astype(str)
df['tipide'] = df['tipide'].astype(str)

# Agregar la columna 'KeyHistorico'
df['KeyHistorico'] = df['Periodo'] + df['tipide'] + df['Documento'].astype(str)

df_final = convertir_columnas_mayusculas(df)

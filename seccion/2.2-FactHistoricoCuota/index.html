    2.2 Fact Historico Cuota¶

    Introducción¶

    Este proceso ETL (Extracción, Transformación y Carga) tiene como objetivo principal centralizar la información histórica de cuotas asociadas a giros de subsidios. Primero, se configura el entorno mediante la importación de librerías y funciones personalizadas para establecer conexiones con las bases de datos y manejar el almacenamiento y la transformación de datos. El script se conecta a la base de datos Minerva para obtener datos de giros y cuotas desde la tabla subsi09, a la cual se le aplica un LEFT JOIN con la tabla subsi22 para integrar datos adicionales de beneficiarios. Luego, la información extraída se organiza y clasifica según el tipo de subsidio, la oportunidad de pago y otras variables relevantes.

    Posteriormente, los datos se enriquecen en el Data Warehouse (DWH) con claves únicas, identificadores secuenciales y etiquetas de clasificación, como TipoGiro, KeyHistoricos, y KeyEstadoGiroCuota, lo cual facilita la administración y análisis de la información. Al final, el DataFrame resultante se carga en la tabla BD_Fact_HistoricoCuota en el DWH, reemplazando los datos previos si existen, para mantener una fuente de datos consolidada y actualizada sobre el estado de giros y cuotas de subsidios.

    Importación de librerías y funciones¶

    Este script configura un entorno para la conexión y manipulación de datos en una base de datos, empleando SQLAlchemy y Pandas junto a un conjunto de funciones personalizadas. Al inicio, establece un contador de tiempo usando time, y prepara el uso de las librerías sqlalchemy, pandas, numpy, y logging para tareas de análisis y registro de eventos. A través de la manipulación de rutas en sys.path, permite la importación de funciones desde el archivo Funciones.py, como guardar_en_dwh, cargar_tablas, obtener_conexion, testfunciones, y setup_logger, que proporcionan métodos específicos para la conexión con la base de datos, la carga de datos y el manejo de eventos, estructurando el código para optimizar las operaciones de almacenamiento y procesamiento de datos.

    Conexión y Procesamiento de Datos en Base Minerva¶

    El código se conecta a la base de datos Minerva mediante SQLAlchemy, usando un motor (motor_minerva) que obtiene su conexión a través de la función obtener_conexion. Una vez establecida la conexión, se ejecuta cargar_tablas, función encargada de extraer los datos especificados en la consulta qr_structure_s9 y almacenarlos en df_structure_s9. Posteriormente, los datos resultantes (df_final_s9) se almacenan en el Data Warehouse (DWH) en la tabla subsi09, reemplazando cualquier contenido previo si existe.

    El código también define otra consulta SQL en qr_structure que, partiendo de la tabla subsi09 en el DWH, clasifica el campo tipsub en "Rural", "No definido", o "Urbano" según condiciones específicas. Además, la consulta evalúa el campo Oportunidad para etiquetar registros como "Oportuno" o "No oportuno" en función de la comparación entre numgir y numenv de la tabla Oportunidad, unida mediante un LEFT JOIN.

    #Lista de querys
qr_structure_s9 = {
    "query": """
select c.id, b.pergir, b.cedres, b.codben, b.numgir, b.numcuo, b.valor, b.tipgir, b.fecasi, b.fecent, b.codcue, b.tipsub
from subsidio.subsi09 as b
left join (select CONCAT(coddoc, documento) as id, codben from subsidio.subsi22) as c
on b.codben = c.codben 

where STR_TO_DATE(CONCAT(pergir, '01'), '%Y%m%d') <= CURRENT_DATE()
and STR_TO_DATE(CONCAT(pergir, '01'), '%Y%m%d') >= cast(DATE_SUB(CURRENT_DATE(), INTERVAL 18 MONTH) as CHAR)
"""
}

    select

    Transformación de Datos y Preparación de Columnas en df_final¶

    Este código aplica una serie de transformaciones en el DataFrame df_final para estructurar los datos de manera uniforme y estandarizada. Cambia el tipo de datos de la columna codben a texto, renombra pergir como periodo, y agrega nuevas columnas como KeyHistoricos, formada a partir de la concatenación de periodo, codben y el sufijo 'Ben'. Además, genera TipoGiro para clasificar el tipo de giro en "Adicional" o "Normal" según el valor de tipgir, y KeyEstadoGiroCuota para crear identificadores únicos. Tras convertir los nombres de todas las columnas a mayúsculas, se renombran campos clave (por ejemplo, ID como ID_AFILIADO, VALOR como VALOR_CUOTA) y se agrega ID_REGISTRO como un identificador secuencial para cada fila. Finalmente, se reorganizan las columnas en un orden específico para una estructura consistente, con ID_REGISTRO al inicio, seguido de los campos principales y claves generadas. Este flujo asegura que el DataFrame esté optimizado para cargas posteriores y análisis en bases de datos u otros sistemas.

    # Asumiendo que df_final = df_structure['query']
# Crear la columna 'Validador' basado en la comparación entre 'fecafi' y 'fecapr'

df_final = df_structure['query']
# Cambiar el tipo de columna 'codben' a texto
df_final['codben'] = df_final['codben'].astype(str)

# Renombrar la columna 'pergir' a 'periodo'
df_final = df_final.rename(columns={'pergir': 'periodo'})

# Agregar la columna 'KeyHistoricos'
df_final['KeyHistoricos'] = df_final['periodo'] + df_final['codben'] + 'Ben'  # Equivalente a Text.Start('Beneficiario',3)

# Agregar la columna 'TipoGiro'
df_final['TipoGiro'] = df_final['tipgir'].apply(lambda x: 'Adicional' if x == 'A' else 'Normal')

# Agregar la columna 'KeyEstadoGiroCuota'
df_final['KeyEstadoGiroCuota'] = df_final['periodo'] + df_final['cedres']

    # Convertir todos los nombres de las columnas de df_final a mayúsculas
df_final.columns = df_final.columns.str.upper()

# Cambiar el nombre de dos campos específicos
df_final = df_final.rename(columns={

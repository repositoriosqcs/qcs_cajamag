    1.5 Fact Datos Contacto¶

    Introducción¶

    En este proceso ETL se extraen y transforman datos de tablas relacionadas con información de contacto y datos bancarios de afiliados y beneficiarios, para su carga final en una tabla de hechos en el Data Warehouse (DWH), denominada BD_Fact_Datos_Contactos. Las tablas de origen incluyen xml4c086, que contiene datos sobre afiliados, xml4c085, que maneja información de sucursales y empresas, y xml4c087, que presenta datos sobre los beneficiarios. La información obtenida de estas tablas cubre datos de contacto como teléfono, correo electrónico y dirección, además de detalles sobre el estado civil, ubicación y cuentas bancarias de los registros procesados.

    La salida final, BD_Fact_Datos_Contactos, almacena estos datos, permitiendo el análisis de los puntos de contacto y detalles bancarios para cada afiliado o beneficiario. Este proceso aplica limpieza de duplicados, estandarización de direcciones y transforma las fechas a un formato trimestral, optimizando los datos para su consulta y análisis.

    Validador de Campos Repetidos y Limpieza de Duplicados¶

    Este código valida y limpia registros duplicados en todas las tablas dentro de df_structure. Para cada tabla, verifica si existe una columna id; si no, intenta crearla concatenando numdoc y coddoc, o registra un error en caso de no encontrar esas columnas. Después de asegurar la existencia de id, se definen ColumnsToCompare como el conjunto de columnas a comparar, excluyendo id, y se procede a almacenar duplicados utilizando StoreDuplicated, que guarda los registros duplicados en un archivo CSV con el prefijo trazaDuplicados_.

    A continuación, RemoveDuplicated elimina los duplicados en cada tabla, conservando el registro más reciente basado en la columna fecest. El tiempo total de validación y limpieza se registra en el logger, proporcionando un resumen del tiempo de procesamiento de este validador de duplicados.

    # Validador de campos repetidos
validador_time = time.time()

# Iterar sobre todas las tablas
for ky in [x for x in df_structure.keys()]:
    df = df_structure[ky]

    # Validar si la columna 'id' existe; si no, crearla
    if 'id' not in df.columns:
        if 'numdoc' in df.columns and 'coddoc' in df.columns:
            # Crear la columna 'id' concatenando 'numdoc' y 'coddoc'
            df = df.assign(id=df['numdoc'].astype(str) + df['coddoc'].astype(str))
            logger.info(f"Columna 'id' creada en el DataFrame: {ky}")
        else:
            # Si no existen las columnas necesarias para crear 'id', saltar la tabla
            logger.error(f"No se encontraron las columnas 'numdoc' y 'coddoc' necesarias para crear 'id' en el DataFrame: {ky}")
            continue  # Saltar a la siguiente tabla si no se pueden crear los valores de 'id'

    # Asegurarse de que la columna 'id' existe antes de proceder
    if 'id' in df.columns:
        # Definir las columnas para comparar, excluyendo 'id'
        ColumnsToCompare = [x for x in df.columns if x != 'id']

        # Imprimir el nombre de la tabla en proceso
        print(f"Procesando tabla: {ky}")

        # Almacenar duplicados
        try:
            StoreDuplicated('id', ColumnsToCompare, df, f'trazaDuplicados_{ky}')
        except KeyError as e:
            logger.error(f"Error en StoreDuplicated para la tabla {ky}: {str(e)}")
            continue

        # Limpieza inicial para quitar duplicados
        try:
            df_structure[ky] = RemoveDuplicated('id', 'fecest', df)
        except KeyError as e:
            logger.error(f"Error en RemoveDuplicated para la tabla {ky}: {str(e)}")
            continue

        # Registrar la validación en el log
        logger.info(f'VALIDADOR TABLA: {ky}')
    else:
        logger.error(f"La columna 'id' no fue creada correctamente en el DataFrame: {ky}")

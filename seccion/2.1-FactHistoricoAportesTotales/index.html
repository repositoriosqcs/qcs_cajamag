    2.1 Fact Historico Aportes Totales¶

    Introducción¶

    Este proceso ETL extrae, transforma y carga datos relacionados con los aportes empresariales en la tabla FactHistoricoAportesTotales. La extracción se realiza desde las tablas subsidio.subsi11 y BD_Dim_Datos_Fijos, que contienen información sobre fechas de pago (fecpag), confirmación (feccon), referencia de periodo (periodorefer), identificación de empresa (nit), valor del aporte (valorpag), valor consignado (valorcon), interés asociado (valorInt) y trabajadores afiliados (trabaapo). Durante la transformación, se manipulan columnas, se agregan claves únicas y se realiza la limpieza de duplicados, estructurando los datos para integrarlos en el Data Warehouse (DWH).

    Definición de Consulta SQL para Datos de Aportes Empresariales¶

    Este código define una consulta SQL en qr_structure que agrupa y resume información de pagos de aportes de la tabla subsidio.subsi11. La consulta selecciona los campos fecpag (fecha de pago), feccon (fecha de confirmación), periodorefer (periodo de referencia) y nit (identificación de la empresa), sumando valores de contribuciones (valorcon), pagos (valorpag), trabajadores aportantes (trabaapo), y valores de interés (valorInt). Los datos se filtran para incluir solo registros de los últimos 18 meses, desde la fecha actual hasta la fecha indicada por DATE_SUB. Además, la consulta clasifica cada registro bajo el tipo 'Empresa' para identificar su origen.

    El resultado se almacena en df_structure y queda listo para ser cargado en un DataFrame para su posterior procesamiento o almacenamiento, lo cual facilita el análisis de aportes empresariales en el periodo especificado.

    #Lista de querys
qr_structure = {
    "query": """ SELECT fecpag, feccon, periodo as periodorefer, nit, 
        SUM(valcon) as valorcon, 
        SUM(valpag) as valorpag, 
        SUM(traapo) as trabaapo, 
        'Empresa' as Tipo, 
        SUM(valint) as valorInt 
    FROM subsidio.subsi11
    WHERE fecpag >= cast(DATE_SUB(CURRENT_DATE(), INTERVAL 18 MONTH) as CHAR)
    AND fecpag <= CURRENT_DATE()
    GROUP BY fecpag, nit;
    """
}
df_structure = dict()
logger.info('LECTURA DE QUERYS')

    Transformación de Datos para Análisis de Aportes Empresariales¶

    Este bloque de código lleva a cabo una transformación detallada de los datos de aportes empresariales almacenados en df. Después de ejecutar una consulta SQL para cargar los datos de aportes (subsidio.subsi11), todas las columnas de df se convierten a mayúsculas y algunas se renombraron para mejorar la claridad (como FECPAG a FECHA_PAGO_APORTE y VALORCON a VALOR_CONSIGNADO). También se crea una columna ID_REGISTRO que asigna un identificador secuencial a cada fila.

    Adicionalmente, una consulta en qr_structure_fijos extrae los datos fijos de la tabla BD_Dim_Datos_Fijos en el data warehouse. Este resultado se almacena en df_fijos, y luego de eliminar duplicados en el campo NIT, se realiza un inner join entre df y df_fijos para agregar los datos fijos relevantes basándose en el campo NIT.

    Finalmente, el DataFrame resultante se reorganiza según el orden en columnas_ordenadas para obtener un conjunto de datos estructurado y listo para análisis de aportes, que permite evaluar la información de pago, contribuciones, y referencia temporal de los aportes empresariales.

    1.7 Fact Historico Trabajadores 2¶

    Introducción¶

    En este proceso ETL se extraen y transforman datos relacionados con trabajadores para almacenarlos en una tabla de hechos en el Data Warehouse (DWH). Los datos se obtienen desde la tabla BD_Fact_op_Trabajadores_p1 y se transforman para proporcionar información consolidada sobre los trabajadores, como identificadores únicos, documentos, estados civiles, períodos de afiliación y retiro, y detalles adicionales relevantes.

    La salida final, BD_Fact_op_Trabajadores_p3, contiene información estandarizada y enriquecida, permitiendo un análisis detallado de la historia laboral de cada afiliado. Este proceso incluye la estandarización de los nombres de columnas, la clasificación de trabajadores según su ubicación (rural o urbano) y la identificación de eventos clave como afiliaciones y retiros en el período especificado.

    ETL¶

    Importación de librerías y funciones¶

    Este script inicializa el entorno para cargar, transformar y almacenar datos utilizando SQLAlchemy, Pandas y funciones personalizadas definidas en un módulo externo (Funciones.py). La biblioteca SQLAlchemy facilita la creación de conexiones y el manejo de transacciones en bases de datos relacionales mediante create_engine, mientras que Pandas permite la manipulación de datos en formato tabular, ideal para tareas de carga y transformación. La biblioteca logging permite configurar un sistema de registros, esencial para monitorear el estado y los posibles errores durante la ejecución.

    El script extiende el sys.path para incluir el directorio donde se encuentran las funciones personalizadas, lo cual permite importar funciones clave como convertir_columnas_mayusculas (para estandarizar nombres de columnas), guardar_en_dwh (para almacenar datos en un Data Warehouse), cargar_tablas (para cargar datos desde diversas fuentes), obtener_conexion (para establecer una conexión a la base de datos), testfunciones (para pruebas) y setup_logger (para configurar los registros). Esta estructura modular y organizada permite un flujo de trabajo de ETL o análisis de datos que es dinámico y escalable.

    Extracción y Transformación de Datos de Trabajadores en el Data Warehouse¶

    Este código define una consulta SQL (op_Trabajadores_p3) para extraer datos de trabajadores almacenados en la tabla BD_Fact_op_Trabajadores_p1 dentro del Data Warehouse (DWH). La consulta selecciona varios campos relevantes, como CODIGO_DOCUMENTO, ID, COD, PERRET, ESTADO, PERIODO, CODCAT, CODIGO_ZONA, CALIFICACION_SUCURSAL, TIPAFI, SEXO, ESTCIV, NIVEDU y FECNAC. Además, incorpora transformaciones para clasificar a los trabajadores como "Rural" o "Urbano" según el campo RURAL, y marca si un trabajador se retiró (SE_RETIRO) o se afilió (SE_AFILIO) en el período especificado.

    La consulta se ejecuta mediante cargar_tablas, que carga los datos desde qr_structure2 en el DataFrame df_structure2 utilizando una conexión SQLAlchemy (motor) a la base de datos del DWH. Después de extraer los datos, el DataFrame se estandariza usando convertir_columnas_mayusculas, lo que asegura que los nombres de las columnas tengan un formato consistente. Finalmente, df_standar.columns.tolist() imprime los nombres de las columnas, permitiendo una rápida verificación de los cambios realizados antes de continuar con el procesamiento o análisis de datos adicionales.

    #Lista de querys

                IF( RURAL = 2 , "Rural" , "Urbano" ) as RURAL,

    Almacenamiento de Datos Estandarizados en el Data Warehouse¶

    Este último paso guarda el DataFrame df_standar en el Data Warehouse dentro de la tabla BD_Fact_op_Trabajadores_p3. La función guardar_en_dwh se encarga de realizar esta operación, donde el parámetro if_exists='replace' indica que la tabla será reemplazada si ya existe, garantizando que los datos almacenados estén actualizados. El parámetro multiple=False optimiza la operación de inserción en una única transacción. Además, el uso del logger permite registrar esta operación, lo que facilita el monitoreo del proceso de carga y asegura la trazabilidad. Este flujo garantiza que los datos estandarizados y transformados estén disponibles en el DWH para futuros análisis o reportes.

    guardar_en_dwh(df_standar, 'BD_Fact_op_Trabajadores_p3', logger, multiple=False, if_exists='replace')

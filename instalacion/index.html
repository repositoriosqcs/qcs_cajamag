    Instalación y Configuración del Entorno para el Procesamiento de Datos del Data Warehouse¶

    Introducción¶

    Este documento presenta los pasos necesarios para configurar un entorno que permita la instalación y configuración de todas las dependencias necesarias para ejecutar procesos de carga y transformación de datos dentro de un Data Warehouse (DWH). Además, se detalla el uso de las funciones contenidas en Funciones.py y cómo se aplican en el flujo de trabajo descrito en los notebook.

    Librerías Necesarias¶

    Para ejecutar correctamente el proyecto, es necesario instalar las siguientes librerías de Python:

    • sqlalchemy version 1.4.39: Para la conexión con bases de datos.
    • pandas version 2.0.3: Para la manipulación y análisis de datos.
    • beautifulsoup4 4.12.2: Para la limpieza de datos en formato HTML.
    • concurrent.futures: Para la paralelización de tareas.
    • numpy version 1.24.3: Para el manejo de arreglos y cálculos numéricos.
    • logging: Para el registro de actividades y errores en el proceso ETL.
    • os: Para la gestión del sistema operativo y rutas de archivos.
    • sys version 3.11.5: Para modificar el sys.path y permitir la importación de módulos personalizados.
    • datetime: Para el manejo de fechas y tiempos.
    • dateutil.relativedelta: Para operaciones avanzadas con fechas.

    Configuración del Entorno¶

    Para configurar el entorno, se recomienda usar un entorno virtual que permita mantener las dependencias organizadas y aisladas de otros proyectos. Los pasos para instalar las dependencias son los siguientes:

    Creación de Entorno Virtual¶

    Primero, cree un entorno virtual:

    python -m venv etl_env

    Luego, active el entorno:

    • En Windows:
    etl_env\Scripts\activate
    • En Linux/macOS:
    source etl_env/bin/activate

    Instalación de Dependencias¶

    Una vez que el entorno esté activado, instale las dependencias:

    pip install sqlalchemy pandas beautifulsoup4 numpy logging python-dateutil concurrent.futures

    Importación de Funciones desde Funciones.py¶

    El proyecto utiliza funciones definidas externamente en el archivo Funciones.py. Este archivo incluye un conjunto de funciones personalizadas que facilitan la conexión a las bases de datos, la transformación de datos y el almacenamiento en el Data Warehouse. Algunas de las funciones principales contenidas en Funciones.py son:

    1. guardar_en_dwh(df, tabla, logger, multiple, if_exists):¶

    • Propósito: Guarda un DataFrame en una tabla específica del Data Warehouse.
    • Argumentos: Permite especificar la tabla, el logger para el registro del proceso y si los datos se deben agregar o reemplazar.

    2. obtener_conexion(nombre_bd):¶

    • Propósito: Proporciona la cadena de conexión para conectarse a una base de datos específica.
    • Uso: Utilizado para establecer la conexión con las bases de datos.

    3. setup_logger(log_filename, log_level):¶

    • Propósito: Configura el logger para registrar eventos y errores durante la ejecución del proceso ETL.
    • Argumentos: El nombre del archivo de log y el nivel de detalle (INFO, DEBUG, etc.).

    4. limpiar_html(texto_html):¶

    • Propósito: Utiliza BeautifulSoup para limpiar el texto HTML, extrayendo únicamente el contenido textual y eliminando etiquetas HTML.

    5. log_tiempo(logger, mensaje, tiempo):¶

    • Propósito: Registra el tiempo transcurrido para una operación específica en el proceso ETL.
    • Argumentos: Recibe el logger, un mensaje descriptivo y el tiempo transcurrido.

    6. cargar_tablas(motor, queries, df_structure, logger):¶

    • Propósito: Carga las tablas especificadas en un diccionario de consultas SQL y las almacena en una estructura de datos.
    • Argumentos: Incluye el motor de conexión, las consultas SQL, un diccionario para almacenar los resultados y el logger para registrar el proceso.

    7. StoreDuplicated(nombre_archivo, columnas, dataframe, ruta):¶

    • Propósito: Almacena registros duplicados en un archivo para su análisis posterior.
    • Argumentos: Recibe el nombre del archivo, las columnas a comparar, el DataFrame y la ruta donde guardar los registros duplicados.

    8. testfunciones():¶

    • Propósito: Verifica que todas las funciones importadas desde Funciones.py están operativas.
    • Uso: Se ejecuta al inicio del script para garantizar la correcta importación de las funciones.

    9. estandarizar_direccion(direccion):¶

    • Propósito: Estandariza el formato de una dirección para asegurar la uniformidad de los datos.
    • Argumentos: Recibe la dirección a estandarizar.

    10. marcar_direcciones_estandarizadas(df, columna):¶

    - **Propósito**: Marca las direcciones estandarizadas dentro de un DataFrame.
- **Argumentos**: Recibe el `DataFrame` y la columna de direcciones a analizar.

    Estas funciones permiten simplificar y automatizar las tareas comunes de un proceso ETL, garantizando un flujo de trabajo eficiente y bien documentado.

    Configuración del Entorno de Trabajo¶

    Configuración Inicial del Logger¶

    La configuración del logger es fundamental para garantizar el seguimiento de las operaciones que se realizan en el proceso ETL. Utilice la función setup_logger para registrar los eventos.

    import logging
from Funciones import setup_logger

logger = setup_logger(log_filename='etl_proceso.log', log_level=logging.INFO)
logger.info('Inicio del proceso ETL')

    Ejecución de Funciones de Conexión y Procesamiento¶

    A continuación, se muestra un ejemplo de cómo utilizar las funciones para establecer una conexión con el DWH y procesar los datos:

    from Funciones import obtener_conexion, guardar_en_dwh
import pandas as pd
from sqlalchemy import create_engine

# Establecer conexión al DWH
cadena_conexion = obtener_conexion('dwh')
motor = create_engine(cadena_conexion)

# Procesar y guardar los datos
df = pd.read_sql_query("SELECT * FROM fuente_datos", motor)

# Guardar en DWH
guardar_en_dwh(df, 'tabla_destino', logger, multiple=False, if_exists='append')

    Conclusión¶

    La correcta configuración del entorno y la instalación de dependencias son fundamentales para asegurar que los procesos de carga y transformación de datos se realicen de manera eficiente y sin contratiempos. Al tener un entorno adecuadamente configurado, se minimizan los errores y se facilita la depuración y mantenimiento del código. Además, el uso de funciones centralizadas en Funciones.py permite simplificar la lógica de conexión, limpieza y carga de datos, asegurando la reproducibilidad de los procesos y manteniendo un código más ordenado, estructurado y fácil de entender. Esto se traduce en una mayor eficiencia, un menor costo de mantenimiento y una mejor calidad general de los datos procesados, facilitando así la toma de decisiones basada en datos confiables y bien organizados.
